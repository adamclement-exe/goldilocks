Scrum & Kanban Basics

Key Concepts

  • Visualize Workflow: Use the Kanban board to see work move through stages (Ready, In Progress, Testing, Done).
  • Limit Work-In-Progress (WIP): Set maximums for 'In Progress' and 'Testing' columns. This prevents bottlenecks, improves focus, and encourages finishing work ('Stop Starting, Start Finishing'). Exceeding WIP limits often slows down the entire system.
  • Manage Flow: Aim for a smooth, steady flow of work through the board. Identify and address bottlenecks (where work piles up). Track metrics like Cycle Time.
  • Make Policies Explicit: Understand the rules (like WIP limits, how blockers are handled, Definition of Done).
  • Implement Feedback Loops: Use Daily Scrums, Sprint Reviews, and Retrospectives to inspect and adapt.
  • Improve Collaboratively: Use feedback to make changes for the next Sprint.

Game Specifics

  • WIP Limits: Displayed on column headers (e.g., "WIP: 1/2"). Assigning work will be blocked if a limit is reached.
  • WIP Aging: Shows how many days a card has been in 'In Progress' or 'Testing' (e.g., "Age: 2d"). High age might indicate a problem or bottleneck.
  • Cycle Time: Calculated in Sprint Review. Measures the average time from starting development ('In Progress') to finishing testing ('Done'). Lower is generally better.
  • Blockers: Obstacles can block stories (indicated by üüß border/icon). Blocked stories cannot progress and increase WIP age. Assign a Senior Developer during the Reassignment phase to unblock a story (costs points!).
